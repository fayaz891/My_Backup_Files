sass selecter functions

seleter is basically CSS selecters(i.e id,classes,tag name etc) which are manupulate with sass functions e,g .abc a ,#ab li etc
selecter functions are used in varaibles e.g $selector:selector-nested("ul","li"); it nested these 2selectors ,now this varaible is used in class by interpolation functions ,#{$selector}{width:10px;} , it return ul li{width:10px;}
only the 1st 3 slector functions are used in any projects, not all these mostly

1)selector-nest() =>it is used to make parent and child selector,1st parameter parent, 2nd is child
e.g $selector:selector-nested(".abc,.xyz","p") then interpolate it #{$selector}{width:20px;} now it return .abc p,.xyz p{width:20px;}
selector-nested(".abc","&:hover") return .abc:hover{}

2)selector-append() =>it is used to combine 2 selectors e.g $selector:selector-append("a",".active") after interpolation it return a.active{..}
it combine 2 selectors while selector-nested separate 2 selecters which are present in their parameters other example (".abc",_copy,_image) return .abc_copy,.abc_image

3)selector-replace() =>it is used to replace selector,it has 3 parameters ("class in which replace","on which replace","new value") e.g $selector:selector-replace("a.abc",".abc","link") it return a.link{..}, it remove the 2nd parameter and and add third parameter

4)is-superselector() =>it is used just to test,it return true or false .it has 2 parameters("superselector","subselector"), it find superselecter in subseleter
e.g is-superselector("a","a.active") it return true bc a is present in a.active ,it is basically used with ifConditions

5)simple-slectors() =>it seperate 2 or more selectors e.g simple-selectors("a.abc"); it return a,.abc{...}

6)selector-unify()=>it is used to match 2 selectors if possible,if not possible then nothing will return
e.g selector-unify("a",".active") it return a.active{..} ,if selector-unify("a","h2") it return nothing

7)selector-extend()=> it just like unify selector but there is some difference which is very complicted,it is not used proffisionlly

sass map functions

it is just like list functions,we can store multiple values in a varaibles just like list, but on other method e.g $map:("red":#ff0000,"green":#00ff00);

$light-weight:("lightest":100,"light":300)

1)map-get()=> it is used to get a value of a key e.g font-weight:map-get($fontWeight,"medium"). it has 2 parameters (varaible,key) . it return font-weight:500;

2)map-merge()=> it is used to combine to different maps,it is not used as others, 1st we make a map of 2 different maps i.e $merge:map-marge($fontWeight,$light-weight);
then it is used with map-keys or map-values etc. now font-weight:map-keys($merge); ,now it return both varaible keys in single map

3)map-remove()=> it is also used as map-merge,it remove keys or values, it is used with map-keys or map-values ,procedure as above

4)map-keys()=>it is used to show all keys which is in varaible e.g font-weight:map-keys($fontWeight); it return font-weight:"regular","medium","bold"

5)map-values()=> it is just like map-keys, it return all values of keys e.g font-weight:map-values($fontWeight); it return font-weight:400,500,700;

6)map-has-key()=> it is used to find a key in varaible, it return true or false e.g font-weight:map-has-key($fontWeight,"bold"), it return true,
